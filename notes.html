Lesson 1: The Basics of the Web and HTML

Components

  • HTML - the main type of document on the web
  • HTTP - the main protocol of the web
  • Tag - an HTML tag is always contained within angled brackets. Most tags have an opening tag (<p> for example) and a closing tag, (</p>). Some tags (called "void" tags) do not require a closing tag (like the <br> tag). Examples:
    • <em> - italics
    • <a> - anchor, used for making links
    • <image> - void tag, has no content. Images have no content, they do not need a closing tag (note: anchor tags use the href attribute while img tags use the src attribute)
    • <br> - also a void tag, creates a line break
    • <p> - paragraph, makes a block. Makes a box around the text. Can have width and height. Different from <br>, which will just create a new line and should be used "inline"
  • Element - an HTML element refers to everything within a set of opening and closing tags.
    • Container Elements: elements that do nothing but contain content inside them.
      • <span> - use inline, for a small amount of content
      • <div> - "division element", use to create a block (with line breaks)
  • Attribute - this is a property of an HTML element. For example, to set the href attribute of an anchor tag to the Udacity URL, you would write <a href="www.udacity.com">

Lesson 2: Creating a Structured Document with HTML

Overview of lesson:

  1. How the web works
  2. How HTML is structured
  3. How web browsers interpret that structure

How the web works

Three languages of web development: HTML, CSS, JavaScript (though no JavaScript in this ND). They each have different purposes when developing a website.

How HTML is structured

HTML controls the "structure" of a web page, analogy: rooms in a house. CSS controls the "style" of a page (how it looks), analogy: color of room walls in a house.

How web browsers interpret that structure

When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like (or branching) structure of a page. The browser builds the DOM tree.

How does the browser know what the structure is? Because HTML classifies elements -- what is text, what is an image, a browser can understand structure. Remember that content is what is inside element tags.

Boxes are used to divide up the structure of a webpage. *everything* on a webpage is a box, even circles. For example, unchecking border-radius property changes circles into squares -- they are circles in the first place due to styling.
These boxes are GREAT. They allow for easier page construction (both design and creation)

Translating a design into a web page.
  1. Create some boxes
  2. Create divs for the outer-most/biggest boxes
  3. Create nested divs for the nested boxes

The way nested boxes thus look in the end is indented html elements, like bullets and sub-bullets in a list.

Lesson 3: Adding CSS Style and HTML Structure

Goals

  1. Add style with CSS
  2. Learn to avoid repetition

What is CSS?

CSS stands for "Cascading Style Sheets."
CSS is code written to control the "style" of HTML elements.
CSS allows programmers to avoid repetition.

Find and Replace

You can also think about CSS as a find and replace tool: you identify a class or a tag of the element you want to find (or match, in CSS terminology), and then what you want to do with it, or what property values to replace with different ones (rather than repeating the same styling for each area).

PEMDAS, AKA Order Matters

It also matters where you define the rules and in what order they are applied, later rules can end up overwriting earlier ones. Types of rules:

  • the default style of a browser (different browsers have slightly different styles)
  • stylesheet in a separate file (this is what you will be mostly using)
  • stylesheet inside HTML (this can be done for small projects but is not ideal)
  • inline style in an element (this can also be done but should be avoided)

Inheritance

"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

How to apply CSS?

Three main ways.
  1. Write CSS in the <head> of your HTML: good for very small projects.
  2. Link your HTML to a separate CSS file: good for majority of projects.
  3. Write your style inline with your HTML: not good, forces lots of repetition. (avoid this to save time, avoid errors and keep consistent, keeping the page looking nice)
